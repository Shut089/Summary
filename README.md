# План автоматизации тестирования  
## Запись на курс «Инженер по тестированию» (Netology)

---

## 1. Цель документа

Документ описывает план автоматизации тестирования пользовательского сценария записи на курс **«Инженер по тестированию»** на сайте Нетологии.

⚠️ В рамках планирования предполагается проверка логики заполнения и отправки формы,  
фактическая отправка формы на реальном сайте **не выполняется**.

---

## 2. Валидные входные данные

Используются во всех позитивных сценариях.

- **Имя**: строка из букв, длина 2–30 символов  
  Пример: `Антон`
- **Телефон**: номер РФ в формате `+7XXXXXXXXXX`  
  Пример: `+79991234567`
- **Email**: корректный email  
  Пример: `test.user@example.com`
- **Чекбокс согласия**: отмечен

---

## 3. Перечень автоматизируемых сценариев

### Навигация к странице курса

### Сценарий 1  
**Название:** Переход к странице курса через меню «Каталог курсов»  
**Данные:** не требуются  

**Шаги:**
1. Открыть главную страницу сайта Нетологии.
2. Нажать «Каталог курсов».
3. Выбрать направление «Программирование».
4. Выбрать курс «Инженер по тестированию».

**Ожидаемый результат:**  
Открыта страница курса «Инженер по тестированию».

---

### Сценарий 2  
**Название:** Переход к странице курса через поиск по каталогу  
**Данные:** строка поиска `Инженер по тестированию`

**Шаги:**
1. Открыть главную страницу.
2. Нажать «Каталог курсов».
3. Ввести название курса в строку поиска.
4. Выбрать курс из результатов.

**Ожидаемый результат:**  
Открыта страница курса «Инженер по тестированию».

## Позитивный сценарий отправки формы

### Сценарий 5  
**Название:** Отправка формы с валидными данными  
**Данные:** валидные данные (см. раздел 2)

**Шаги:**
1. Заполнить все обязательные поля валидными данными.
2. Отметить чекбокс согласия.
3. Нажать кнопку «Записаться».

**Ожидаемый результат:**  
Форма успешно отправлена, отображается сообщение об успешной записи / происходит переход на страницу подтверждения.

## Негативные сценарии — обязательные поля

**Название:** Отправка формы с пустым полем «Имя»

## 4. Инструменты автоматизации

- Java + Selenide  
- JUnit 5  
- Gradle  
- Allure (опционально)

## 5. Риски автоматизации

- Реальная отправка заявки.
- Изменения UI и A/B-тестирование.
- Антибот-защита и капча.
- Динамический контент страницы.
